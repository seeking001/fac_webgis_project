# 龙华区公共服务设施规划管理系统

## 概述

本项目范围以深圳市龙华区为核心，融合城市规划业务和编辑技术，将规划专业需求转化为可落地的技术解决方案，旨在构建一个集空间数据管理、可视化分析和交互编辑于一体的规划工具，实现公共服务设施与土地利用的综合管理功能。



## 项目背景

国内城市规划工具经历了从CAD制图到GIS分析的演进过程。传统规划工具存在数据孤岛、协同困难、更新滞后等问题。本项目聚焦于规划信息化建设的思考：

**行业痛点：**

- 规划数据分散在不同部门和格式中，缺乏统一管理
- 空间分析依赖专业软件，决策支持响应慢
- 公众参与渠道有限，规划方案沟通成本高
- 规划成果更新周期长，难以适应快速城市发展

**技术愿景：**

- 构建基于Web的规划管理平台，实现数据集中管理
- 开发直观的可视化界面，降低GIS使用门槛
- 实现实时协同编辑，提升规划编制效率
- 建立开放数据接口，支持多源数据融合



## 技术架构

### 架构设计理念

注重系统的实用性和可维护性，架构设计遵循以下原则：

- **前后端分离**：前端专注用户体验，后端保证数据安全
- **微服务化**：每个服务职责单一，便于扩展和维护
- **空间数据优先**：从数据库设计到API都充分考虑GIS特性
- **渐进式增强**：核心功能优先，高级功能按需加载

### 技术栈选型

```
前端层：
├── Vue3 + Composition API (现代化响应式框架)
├── OpenLayers (专业级地图渲染引擎)
├── Pinia (轻量状态管理)
└── Vite (极速构建工具)

服务层：
├── Node.js + Express (高性能服务端)
├── PostgreSQL + PostGIS (空间数据库)
├── GeoServer (OGC标准地图服务)
└── Docker (容器化部署)

开发工具链：
├── Git + GitHub (版本控制)
├── ESLint + Prettier (代码规范)
└── Postman + pgAdmin (调试工具)
```

## 使用指南

### 1. 地图操作

- **底图切换**: 左上角控件切换普通地图/卫星影像
- **路网显示**: 底图面板中可控制路网图层显示
- **视图控制**: 鼠标滚轮缩放，拖拽平移
- **坐标显示**: 右下角实时显示鼠标所在位置坐标

### 2. 设施管理

1. **加载显示**: 左侧面板勾选"公共设施"
2. **类型筛选**: 下拉选择特定类型设施（学校、医院等）
3. **添加设施**: 点击"绘制添加"，在地图上点击位置，填写信息
4. **编辑位置**: 点击"编辑图形"，拖动设施点调整位置
5. **查看详情**: 点击设施图标查看详细信息
6. **删除设施**: 在详情弹窗中点击删除按钮

### 3. 土地利用

1. **加载显示**: 左侧面板勾选"土地利用"
2. **绘制地块**: 点击"绘制添加"，绘制多边形，填写属性
3. **编辑图形**: 点击"编辑图形"，拖动顶点修改形状
4. **类型筛选**: 按用地类型过滤显示
5. **面积统计**: 右侧面板显示各类用地数量

### 4. 数据交互

- **点击查询**: 点击要素查看详细信息
- **视图筛选**: 仅加载当前视图范围内的要素
- **实时保存**: 编辑操作自动保存到数据库
- **即时显示**: 新增要素立即在地图上显示



## 项目亮点

### 1. 专业与技术融合

- **规划思维**：系统设计源于实际规划工作流程
- **开发实践**：采用现代化Web开发最佳实践
- **GIS深度**：充分利用空间数据库特性

### 2. 性能优化

- **视图动态加载**: 仅加载当前视野范围内的数据
- **图层分级渲染**: 矢量图层按需显示，减少内存占用
- **样式缓存**: 要素样式预先计算，提升渲染性能

### 3. 用户体验

- **即时反馈**: 操作结果立即可视化
- **防误操作**: 编辑模式明确指示，防止误修改
- **数据验证**: 表单输入实时验证，减少无效提交

### 4. 代码质量

- **模块化设计**: 前后端分离，职责清晰
- **错误处理**: 统一的错误处理机制
- **类型安全**: 关键数据接口类型定义

### 5. 可扩展性

- **插件架构**: 新功能可通过模块化方式添加
- **配置驱动**: 地图样式、API端点可配置
- **标准接口**: 遵循OGC标准，易于集成其他GIS服务
